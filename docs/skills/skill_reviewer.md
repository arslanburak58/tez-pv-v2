# SKILL: Eleştirel Tez İncelemesi

## Ne zaman aktif
"Şu paragrafı eleştir", "mantığında boşluk var mı", "tutarlı mı".

## Sevimli olma. Boşluk bul.

## 7 kontrol açısı

1. **Mantıksal akış:** önerme → kanıt → sonuç sağlam mı?
2. **Kanıtsız iddia:** atıf eksik mi?
3. **Tezin özgün ayrışması:** kapasite normalizasyonu / QR meta / multi-plant / robustness / missingness — hangisi bu paragrafta net?
4. **APA tutarlılığı:** format, sayfa numarası gerekli mi?
5. **Türkçe akademik üslup:** gevezelik, fazla sıfat, günlük dil?
6. **Teknik doğruluk:** formül, terim, paket ismi (sklearn küçük 's', QuantileRegressor camelCase)?
7. **Tutarlılık:** tezin diğer bölümleriyle çelişiyor mu?

## v2-Spesifik Ek Kontroller

- "Mutlak kW ile" eğitim ima eden ifade var mı? → HATA, normalize olmalı
- "CQR ile kalibrasyon" geçiyor mu? → v2'de gereksiz
- "Ridge meta-learner" yazıyor mu? → "Lineer quantile regresyon" olmalı
- Sadece DKASC test ediliyor mu? → Holdout PVOD istasyonları belirtilmeli

## Çıktı formatı

**✅ Güçlü yönler** (2-3 madde, gerçekten varsa)
**⚠️ Düzeltilmesi gereken** (cümle bazlı)
**🔧 Önerilen minimal revizyon** (sorunlu cümle için)

- "Genel olarak iyi" yumuşatması
- Tüm metni yeniden yaz

- Kanıtsız eleştiri
